Banco de Dados

Na Matéria de Banco de Dados vimos a importância de utilizar um banco de dados e como armazenar informações e organiza-las em sua etapa inicial.

Para aprender e consolidar o conteúdo foram passados diversos projetos, os quais irei apresentar a seguir.

Modelo Entidade Relacionamento

Nas primeiras aula estudamos o Modelo Entidade Relacionamento (MER), que é uma representação dos requisitos que serão modelados no Banco de Dados. o MER descreve as entidades, os atributos e seus relacionamentos.

Diagrama Entidade Relacionamento

O Diagrama Entidade Relacionamento (DER) tem como objetivo traduzir o que foi feito no MER. o DER é utilizado para mapear as entidades, atributos e relacionamentos para facilitar a construção das tabelas, já com suas colunas e chaves primarias e estrangeiras definidas. Ele também auxilia a eliminar redundâncias e garantir a integridade dos dados.

Modelo Físico

Aprendemos a realizar a construção do modelo físico através do PostgreSQL. è no modelo físico que definimos as estruturas de armazenamento no Banco de Dados, essas estruturas são: as tabelas, índices e restrições de integridade.
Para a construção e manipulação do banco, aprendemos a linguagem SQL (linguagem de consulta estruturada), que consiste em uma linguagem padrão de gerenciamento de dados.
